### 1. What is Apex? (Apex Kya Hai?)

Apex ek **Object-Oriented** programming language hai jo Salesforce ke servers par chalti hai. Iska syntax Java se milta-julta hai.

* **Hosted:** Aapko koi software install nahi karna padta. Code Salesforce ke cloud server par save, compile aur execute hota hai.
* **Strongly-typed:** Apex mein har variable ka type (jaise `Integer`, `String`) fix hota hai. Galat type use karne par code save hi nahi hoga.
* **Multitenant-aware:** Salesforce ek "shared" system hai (jaise ek building mein alag-alag flats). Apex mein **Governor Limits** hoti hain taaki aapka code doosre users ke resources ko slow na kare.
* **Integrated with Database:** Database ke records (Account, Contact) ko access karna aur unhe modify karna bahut easy hai. Isme SQL ki tarah **SOQL** (Search) aur **DML** (Insert/Update) inbuilt hota hai.

---

### 2. Apex Language Highlights (Zaroori Features)

Apex mein woh sab hai jo ek modern language mein hota hai, plus kuch Salesforce-specific cheezein:

* **Common Features:** Classes, Interfaces, Loops (`for`, `while`), aur Conditional statements (`if-then-else`).
* **Triggers:** Yeh special code hote hain jo kisi event (jaise record insert ya delete hone) par apne aap chalte hain.
* **Case-Insensitivity:** Apex mein `String` aur `string` ek hi baat hai. Aap `System.debug` likho ya `system.debug`, code chal jayega.
* **Global Access Modifier:** `public` ke alawa isme `global` hota hai, jo kisi bhi application ya namespace se access kiya ja sakta hai.

---

### 3. Data Types (Data ke Prakaar)

Apex mein data store karne ke liye alag-alag containers hote hain:

1. **Primitives:** * `Integer`: Bina decimal wala number (e.g., 100).
* `Double/Decimal`: Decimal wala number (e.g., 99.50).
* `String`: Text (Humesha single quotes `' '` mein likha jata hai).
* `Boolean`: Sirf `true` ya `false`.
* `ID`: Salesforce ka unique 15 ya 18 character ka record ID.


2. **sObjects:** Yeh sabse important hai. Ek `Account` ya `Contact` record ko Apex variable mein store kiya ja sakta hai.
3. **Collections:** * **Lists:** Ordered list (Duplicate values allow hain).
* **Sets:** Unique values ki list (Duplicate allow nahi hain).
* **Maps:** Key-Value pair (Jaise dictionary).



---

### 4. Apex Collections: List (Detail mein)

List ka use tab hota hai jab humein ek se zyada items ek hi variable mein rakhne hon.

* **Declaration:** `List<String> colors = new List<String>();`
(Yahan `String` bata raha hai ki is list mein sirf text aayega).
* **Adding Elements:** `colors.add('red');`
* **Accessing Elements:** * Index se: `colors[0]` (Pehela item).
* Method se: `colors.get(0)`.


* **Iterating (Loop):** Saare items ek-ek karke dekhne ke liye `for` loop use hota hai:
```java
for(String c : colors) {
    System.debug(c);
}

```



---

### 5. Apex Classes & Methods

Class ek "Template" hai aur Methods uske "Actions" hain.

* **Public vs Private:** `public` method ko bahar se call kar sakte hain, `private` sirf usi class ke andar kaam karta hai.
* **Instance Methods:** Inhe use karne ke liye object banana padta hai:
`EmailManager obj = new EmailManager();`
`obj.sendMail(...);`
* **Static Methods:** Inhe bina object ke seedha class name se call karte hain:
`EmailManager.sendMail(...);`
*(Note: Static methods tab use hote hain jab humein class ke variables ko store nahi karna hota).*

---

### 6. Development Tools (Kahan likhein code?)

1. **VS Code (Salesforce Extensions):** Professional developers ise use karte hain.
2. **Developer Console:** Browser ke andar hi ek mini-tool hai.
* **File > New > Apex Class:** Nayi class banane ke liye.
* **Debug > Open Execute Anonymous:** Bina class save kiye code test karne ke liye.



---

### 7. Inspecting Debug Logs (Output kaise dekhein?)

Jab bhi aap code chalate hain, Salesforce ek "Log" file banata hai.

* **Logs Tab:** Console mein niche hota hai.
* **Debug Only Checkbox:** Is par click karne se system ki faltu ki lines chup jati hain aur sirf aapke `System.debug()` wale messages dikhte hain. Yeh error dhundne ka sabse bada tool hai.

---

### 8. EmailManager Example Flow

Is example mein dikhaya gaya ki kaise ek class banayi jati hai jo email bhejti hai.

* **Messaging.SingleEmailMessage:** Yeh Salesforce ki ek ready-made class hai email set karne ke liye.
* **Messaging.sendEmail:** Yeh final command hai jo email bhejta hai.
* **Error Handling:** `inspectResults` method check karta hai ki email gaya ya nahi, aur result ko console (Debug Log) mein print karta hai.

//sObjects
Salesforce mein **sObjects** ka concept samajhna bahut zaroori hai agar aap Apex coding seekh rahe hain. Iska simple matlab hai: **Salesforce ka har ek record, Apex mein ek Object hota hai.**

Chaliye isse detail mein aur aasaan Hinglish mein samajhte hain:

---

## 1. sObject Kya Hai? (The Basic Concept)

Salesforce mein aap jo bhi data dekhte hain (jaise ek Account, Contact, ya Lead), code ki duniya mein use **sObject** (Salesforce Object) kehte hain.

* **Database Record = sObject:** Agar Salesforce UI mein "Acme" naam ka ek Account hai, toh Apex mein wo `Account` sObject ban jayega.
* **Fields = Properties:** Record ke fields (Phone, Name, etc.) sObject ki properties ban jati hain jinhe aap change kar sakte ho.

---

## 2. sObject Variable Kaise Banayein?

Apex mein record banane ke do tareeke hain:

### A. Constructor Use Karke (Fastest Way)

Aap variable declare karte waqt hi fields ki value assign kar sakte hain.

```java
// Ek naya Account variable banaya aur usi waqt Name aur Phone set kar diya
Account myAcc = new Account(Name='Acme', Phone='(415)555-1212');

```

### B. Dot Notation Use Karke (Step-by-Step)

Pehle empty object banayein, phir ek-ek karke fields bharein.

```java
Account myAcc = new Account();
myAcc.Name = 'Acme';
myAcc.Phone = '(415)555-1212';

```

---

## 3. API Names Ka Chakkar

Apex mein hamesha **API Name** use hota hai, Label nahi.

* **Standard Objects:** Account, Contact, Lead (Inke naam simple hote hain).
* **Custom Objects:** Inke peeche hamesha `__c` lagta hai (e.g., `Merchandise__c`).
* **Custom Fields:** Inke peeche bhi `__c` lagta hai (e.g., `Description__c`).
* **Relationship Fields:** Inke peeche `__r` lagta hai.

> **Note:** Agar field ka label "Employee Name" hai, toh API name "Employee_Name__c" ho sakta hai. Space ki jagah underscore (`_`) aa jata hai.

---

## 4. Generic sObject (Sabka Baap)

Kabhi-kabhi humein pata nahi hota ki kaun sa object aane wala hai (Account ya Contact?). Aise mein hum **Generic sObject** data type use karte hain.

* **Generic sObject** ek aisa variable hai jo kisi bhi Salesforce record ko hold kar sakta hai.

```java
// Generic sObject mein Account bhi aa sakta hai aur Custom Object bhi
sObject obj1 = new Account(Name='Trailhead'); 
sObject obj2 = new Merchandise__c(Name='Laptop');

```

---

## 5. Casting: Generic se Specific Banana

Generic sObject ke sath ek dikkat hai: Aap `obj1.Name` karke field access nahi kar sakte. Iske liye aapko use "Cast" (convert) karna padta hai.

```java
// Ye generic hai
sObject myGeneric = [SELECT Id, Name FROM Account LIMIT 1];

// Dot notation use karne ke liye ise Account mein cast karna padega
Account acc = (Account)myGeneric; 
String accName = acc.Name; // Ab ye kaam karega!

```

## Quiz Answers (For your reference)

1. **Describe the relationship between sObjects and Salesforce records.**
* **Correct Answer: C.** Every record in Salesforce is natively represented as an sObject in Apex. (Har record Apex mein sObject hota hai).


2. **How do you create an sObject instance, such as an Account?**
* **Correct Answer: A.** Declare a variable and assign a new sObject instance to it. (Variable declare karo aur `new` keyword use karo).


3. **Which of the following is correct about a generic sObject variable?**
* **Correct Answer: C.** A generic sObject variable can be assigned to any specific standard or custom sObject... using casting. (Generic variable kisi bhi object ko hold kar sakta hai casting ke zariye).
