### 1. What is Apex? (Apex Kya Hai?)

Apex ek **Object-Oriented** programming language hai jo Salesforce ke servers par chalti hai. Iska syntax Java se milta-julta hai.

* **Hosted:** Aapko koi software install nahi karna padta. Code Salesforce ke cloud server par save, compile aur execute hota hai.
* **Strongly-typed:** Apex mein har variable ka type (jaise `Integer`, `String`) fix hota hai. Galat type use karne par code save hi nahi hoga.
* **Multitenant-aware:** Salesforce ek "shared" system hai (jaise ek building mein alag-alag flats). Apex mein **Governor Limits** hoti hain taaki aapka code doosre users ke resources ko slow na kare.
* **Integrated with Database:** Database ke records (Account, Contact) ko access karna aur unhe modify karna bahut easy hai. Isme SQL ki tarah **SOQL** (Search) aur **DML** (Insert/Update) inbuilt hota hai.

---

### 2. Apex Language Highlights (Zaroori Features)

Apex mein woh sab hai jo ek modern language mein hota hai, plus kuch Salesforce-specific cheezein:

* **Common Features:** Classes, Interfaces, Loops (`for`, `while`), aur Conditional statements (`if-then-else`).
* **Triggers:** Yeh special code hote hain jo kisi event (jaise record insert ya delete hone) par apne aap chalte hain.
* **Case-Insensitivity:** Apex mein `String` aur `string` ek hi baat hai. Aap `System.debug` likho ya `system.debug`, code chal jayega.
* **Global Access Modifier:** `public` ke alawa isme `global` hota hai, jo kisi bhi application ya namespace se access kiya ja sakta hai.

---

### 3. Data Types (Data ke Prakaar)

Apex mein data store karne ke liye alag-alag containers hote hain:

1. **Primitives:** * `Integer`: Bina decimal wala number (e.g., 100).
* `Double/Decimal`: Decimal wala number (e.g., 99.50).
* `String`: Text (Humesha single quotes `' '` mein likha jata hai).
* `Boolean`: Sirf `true` ya `false`.
* `ID`: Salesforce ka unique 15 ya 18 character ka record ID.


2. **sObjects:** Yeh sabse important hai. Ek `Account` ya `Contact` record ko Apex variable mein store kiya ja sakta hai.
3. **Collections:** * **Lists:** Ordered list (Duplicate values allow hain).
* **Sets:** Unique values ki list (Duplicate allow nahi hain).
* **Maps:** Key-Value pair (Jaise dictionary).



---

### 4. Apex Collections: List (Detail mein)

List ka use tab hota hai jab humein ek se zyada items ek hi variable mein rakhne hon.

* **Declaration:** `List<String> colors = new List<String>();`
(Yahan `String` bata raha hai ki is list mein sirf text aayega).
* **Adding Elements:** `colors.add('red');`
* **Accessing Elements:** * Index se: `colors[0]` (Pehela item).
* Method se: `colors.get(0)`.


* **Iterating (Loop):** Saare items ek-ek karke dekhne ke liye `for` loop use hota hai:
```java
for(String c : colors) {
    System.debug(c);
}

```



---

### 5. Apex Classes & Methods

Class ek "Template" hai aur Methods uske "Actions" hain.

* **Public vs Private:** `public` method ko bahar se call kar sakte hain, `private` sirf usi class ke andar kaam karta hai.
* **Instance Methods:** Inhe use karne ke liye object banana padta hai:
`EmailManager obj = new EmailManager();`
`obj.sendMail(...);`
* **Static Methods:** Inhe bina object ke seedha class name se call karte hain:
`EmailManager.sendMail(...);`
*(Note: Static methods tab use hote hain jab humein class ke variables ko store nahi karna hota).*

1. **VS Code (Salesforce Extensions):** Professional developers ise use karte hain.
2. **Developer Console:** Browser ke andar hi ek mini-tool hai.
* **File > New > Apex Class:** Nayi class banane ke liye.

### 7. Inspecting Debug Logs (Output kaise dekhein?)

Jab bhi aap code chalate hain, Salesforce ek "Log" file banata hai.

* **Logs Tab:** Console mein niche hota hai.
* **Debug Only Checkbox:** Is par click karne se system ki faltu ki lines chup jati hain aur sirf aapke `System.debug()` wale messages dikhte hain. Yeh error dhundne ka sabse bada tool hai.

//sObjects
Salesforce mein **sObjects** ka concept samajhna bahut zaroori hai agar aap Apex coding seekh rahe hain. Iska simple matlab hai: **Salesforce ka har ek record, Apex mein ek Object hota hai.**

Chaliye isse detail mein aur aasaan Hinglish mein samajhte hain:

---

## 1. sObject Kya Hai? (The Basic Concept)

Salesforce mein aap jo bhi data dekhte hain (jaise ek Account, Contact, ya Lead), code ki duniya mein use **sObject** (Salesforce Object) kehte hain.

* **Database Record = sObject:** Agar Salesforce UI mein "Acme" naam ka ek Account hai, toh Apex mein wo `Account` sObject ban jayega.
* **Fields = Properties:** Record ke fields (Phone, Name, etc.) sObject ki properties ban jati hain jinhe aap change kar sakte ho.

---

## 2. sObject Variable Kaise Banayein?

Apex mein record banane ke do tareeke hain:

### A. Constructor Use Karke (Fastest Way)

Aap variable declare karte waqt hi fields ki value assign kar sakte hain.

```java
// Ek naya Account variable banaya aur usi waqt Name aur Phone set kar diya
Account myAcc = new Account(Name='Acme', Phone='(415)555-1212');

```

### B. Dot Notation Use Karke (Step-by-Step)

Pehle empty object banayein, phir ek-ek karke fields bharein.

```java
Account myAcc = new Account();
myAcc.Name = 'Acme';
myAcc.Phone = '(415)555-1212';

```

---

## 3. API Names Ka Chakkar

Apex mein hamesha **API Name** use hota hai, Label nahi.

* **Standard Objects:** Account, Contact, Lead (Inke naam simple hote hain).
* **Custom Objects:** Inke peeche hamesha `__c` lagta hai (e.g., `Merchandise__c`).
* **Custom Fields:** Inke peeche bhi `__c` lagta hai (e.g., `Description__c`).
* **Relationship Fields:** Inke peeche `__r` lagta hai.

> **Note:** Agar field ka label "Employee Name" hai, toh API name "Employee_Name__c" ho sakta hai. Space ki jagah underscore (`_`) aa jata hai.

---

## 4. Generic sObject (Sabka Baap)

Kabhi-kabhi humein pata nahi hota ki kaun sa object aane wala hai (Account ya Contact?). Aise mein hum **Generic sObject** data type use karte hain.

* **Generic sObject** ek aisa variable hai jo kisi bhi Salesforce record ko hold kar sakta hai.

```java
// Generic sObject mein Account bhi aa sakta hai aur Custom Object bhi
sObject obj1 = new Account(Name='Trailhead'); 
sObject obj2 = new Merchandise__c(Name='Laptop');

```

---

## 5. Casting: Generic se Specific Banana

Generic sObject ke sath ek dikkat hai: Aap `obj1.Name` karke field access nahi kar sakte. Iske liye aapko use "Cast" (convert) karna padta hai.

```java
// Ye generic hai
sObject myGeneric = [SELECT Id, Name FROM Account LIMIT 1];

// Dot notation use karne ke liye ise Account mein cast karna padega
Account acc = (Account)myGeneric; 
String accName = acc.Name; // Ab ye kaam karega!

```

---

2. **How do you create an sObject instance, such as an Account?**

3. **Which of the following is correct about a generic sObject variable?**
* **Correct Answer: C.** A generic sObject variable can be assigned to any specific standard or custom sObject... using casting. (Generic variable kisi bhi object ko hold kar sakta hai casting ke zariye).

# Salesforce Apex: Manipulating Records with DML (Detailed Notes)

## 1. Introduction to DML (Data Manipulation Language)

Salesforce mein data ko create aur modify karne ke liye **DML** ka use hota hai. Apex ek data-focused language hai, isliye iska direct access Salesforce data par hota hai.

* **Key Advantage:** Dusri languages ki tarah data source se connect karne ke liye extra setup nahi karna padta.

### Basic DML Statements:

1. **`insert`**: Naye records create karne ke liye.
2. **`update`**: Existing records ko modify karne ke liye.
3. **`upsert`**: Naye record create karta hai aur purane ko update (agar match mil jaye).
4. **`delete`**: Records ko delete karne ke liye (Recycle Bin mein jate hain).
5. **`undelete`**: Recycle Bin se records wapas lane ke liye.
6. **`merge`**: Do ya teen same type ke records ko ek banane ke liye (Sirf Account, Contact, Lead aur Case ke liye).

---

## 2. Basic Operation & ID Auto-Assignment

Jab aap koi record `insert` karte ho, toh Salesforce automatically usko ek **Unique ID** assign kar deta hai. Insert hone ke baad aapka sObject variable us ID ko hold kar leta hai.

**Code Example:**

```java
// 1. sObject create kiya
Account acct = new Account(Name='Acme', Phone='(415)555-1212', NumberOfEmployees=100);

// 2. DML statement se insert kiya
insert acct;

// 3. ID automatically populate ho gayi, ab ise use kar sakte hain
ID acctID = acct.Id;
System.debug('ID = ' + acctID);

```

---

## 3. Bulk DML (Most Important)

Salesforce mein kabhi bhi loop ke andar DML nahi karna chahiye (Governor Limits ki wajah se). Hamesha **List** ka use karke bulk operations karne chahiye. Ek transaction mein aap maximum **150 DML statements** hi chala sakte hain.

**Code Example (Bulk Insert & Update):**

```java
// List create ki
List<Contact> conList = new List<Contact> {
    new Contact(FirstName='Joe',LastName='Smith',Department='Finance'),
    new Contact(FirstName='Kathy',LastName='Smith',Department='Technology'),
    new Contact(FirstName='Caroline',LastName='Roth',Department='Finance'),
    new Contact(FirstName='Kim',LastName='Shain',Department='Education')
};

// Bulk insert: Ek hi statement mein 4 records gaye
insert conList;

// List update ke liye
List<Contact> listToUpdate = new List<Contact>();

for(Contact con : conList) {
    if (con.Department == 'Finance') {
        con.Title = 'Financial analyst';
        listToUpdate.add(con);
    }
}

// Bulk update
update listToUpdate;

```

---

## 4. Upsert Operation

Upsert "Update + Insert" ka combo hai. Ye duplicate records se bachata hai.

* **Matching Rule:** Agar record ID hai toh update karega, nahi toh naya banayega. Aap specific "External ID" field bhi de sakte ho matching ke liye.

**Syntax:** `upsert sObjectList [matchingField];`

**Code Example (Using ID and External ID):**

```java
// Part A: Matching via ID
Contact josh = new Contact(FirstName='Josh',LastName='Kaplan',Department='Finance');
insert josh; 

josh.Description = 'Josh\'s record has been updated by the upsert operation.';
Contact kathy = new Contact(FirstName='Kathy',LastName='Brown',Department='Technology');

List<Contact> contacts = new List<Contact> { josh, kathy };
upsert contacts; // Josh update hoga (ID match), Kathy insert hogi

// Part B: Matching via Specific Field (Email)
Contact jane = new Contact(FirstName='Jane', LastName='Smith', Email='jane.smith@example.com', Description='Contact of the day');
insert jane;

Contact jane2 = new Contact(FirstName='Jane', LastName='Smith', Email='jane.smith@example.com', Description='Prefers to be contacted by email.');
// Email field ko use karke upsert kiya
upsert jane2 Contact.fields.Email;

```

---

## 5. Deleting Records

Records ko delete karne ke liye `delete` keyword use hota hai. Deleted records **15 days** tak Recycle Bin mein rehte hain.

**Code Example:**

```java
// Pehle query karke records laye (SOQL)
Contact[] contactsDel = [SELECT Id FROM Contact WHERE LastName='Smith'];
// Delete kiya
delete contactsDel;

```

---

## 6. DML Exceptions (Error Handling)

Agar DML fail hota hai (jaise required field miss hona), toh system `DmlException` throw karta hai. Ise `try-catch` se handle karna chahiye.

**Code Example:**

```java
try {
    Account acct = new Account(); // Name field missing hai (Required)
    insert acct;
} catch (DmlException e) {
    System.debug('A DML exception has occurred: ' + e.getMessage());
}

```

---

## 7. Database Methods (Partial Success)

DML statements (`insert acct;`) "All or None" hote hainâ€”agar ek bhi record fail hua toh pura transaction fail.
Lekin **Database methods** (`Database.insert()`) se aap **Partial Success** allow kar sakte ho.

* **Parameter:** `allOrNone` (Agar `false` hai, toh sahi records save ho jayenge, galat wale error return karenge).

**Code Example (Partial Success):**

```java
List<Contact> conList = new List<Contact> {
    new Contact(FirstName='Joe',LastName='Smith'),
    new Contact() // Error: LastName missing hai
};

// false matlab partial success allowed hai
Database.SaveResult[] srList = Database.insert(conList, false);

for (Database.SaveResult sr : srList) {
    if (sr.isSuccess()) {
        System.debug('Success ID: ' + sr.getId());
    } else {
        for(Database.Error err : sr.getErrors()) {
            System.debug('Error: ' + err.getMessage());
        }
    }
}

```

---

## 8. Working with Related Records

Relationship ke liye hamesha Parent ki ID Child ke field mein dalni padti hai.

### A. Insert Related Records

```java
Account acct = new Account(Name='SFDC Account');
insert acct;
ID acctID = acct.ID;

// Contact ko Account se link kiya
Contact mario = new Contact(FirstName='Mario', LastName='Ruiz', AccountId=acctID);
insert mario;

```

### B. Update Related Records (Separate Calls)

Ek hi DML call mein Parent aur Child update nahi ho sakte. Alag-alag calls karni padengi.

```java
Contact queriedContact = [SELECT Account.Name FROM Contact WHERE FirstName = 'Mario' LIMIT 1];
queriedContact.Phone = '(415)555-1213';
queriedContact.Account.Industry = 'Technology';

update queriedContact;         // 1. Contact update
update queriedContact.Account; // 2. Parent Account update

```

### C. Delete Related Records (Cascading Delete)

Agar aap Parent record delete karte ho, toh uske related Child records automatically delete ho jate hain (agar relationship Master-Detail hai ya cascading enabled hai).

---

## 9. Transactions

Apex mein DML ek **Transaction** ke andar chalta hai.

* Agar ek transaction mein 10 operations hain aur 10th fail ho gaya, toh pehle ke 9 bhi **Rollback** ho jayenge (database mein kuch save nahi hoga).

---

### Comparison Table: DML Statements vs Database Methods

| Feature | DML Statement (`insert`) | Database Method (`Database.insert`) |
| --- | --- | --- |
| **Partial Success** | Not Supported (All or None) | Supported (via `allOrNone` parameter) |
| **Exceptions** | Throws Exception on error | Returns Result Array (no exception if false) |
| **Usage** | Jab aap chahte ho error aane par code ruk jaye | Jab aap error handle karke baki data save karna chahte ho |

# Salesforce Apex: Write SOQL Queries (Detailed Notes)

## 1. SOQL Kya Hai?

SOQL ka matlab hai **Salesforce Object Query Language**. Ye SQL ki tarah hota hai lekin Salesforce platform ke liye customized hai. Iska use Salesforce database se records read karne ke liye kiya jata hai.

* **Inline SOQL:** Jab hum Apex code ke andar hi square brackets `[]` mein query likhte hain, toh use Inline SOQL kehte hain.
* **Syntax:**
```java
Account[] accts = [SELECT Name, Phone FROM Account];

```



---

## 2. Basic SOQL Syntax

Basic query ka structure aisa hota hai:
`SELECT fields FROM ObjectName [WHERE Condition] [ORDER BY Field] [LIMIT n]`

* **SELECT Clause:** Kaunse fields chahiye (jaise Name, Phone).
* **FROM Clause:** Kaunse object se data chahiye (jaise Account ya Custom_Object__c).
* **FIELDS() Keyword:**
* `FIELDS(STANDARD)`: Saare standard fields ke liye.
* `FIELDS(CUSTOM)`: Saare custom fields ke liye.
* `FIELDS(ALL)`: Saare fields ke liye.



> **Note:** Apex queries mein `Id` field hamesha return hoti hai, chahe aap use SELECT mein likho ya nahi. Lekin Query Editor mein dekhne ke liye use likhna padta hai.

---

## 3. Filtering Results (WHERE Clause)

Agar aapko saare records nahi balki kuch specific records chahiye, toh `WHERE` use karein.

* **String Comparison:** Case-insensitive hota hai (`'SFDC'` aur `'sfdc'` same hain).
* **Logical Operators:** `AND`, `OR` aur Parentheses `()` ka use karke complex conditions bana sakte hain.
* **LIKE Operator:** Fuzzy matching ke liye.
* `%`: Any number of characters (jaise `Name LIKE 'SFDC%'`).
* `_`: Exactly one character.



**Code Example:**

```java
// Complex filtering
SELECT Name, Phone FROM Account 
WHERE (Name = 'SFDC Computing' OR (NumberOfEmployees > 25 AND BillingCity = 'Los Angeles'))

```

---

## 4. Ordering and Limiting Results

* **ORDER BY:** Records ko sort karne ke liye.
* `ASC` (Default): Alphabetical/Numerical order.
* `DESC`: Reverse order.


* **LIMIT:** Kitne records return karne hain. Agar `LIMIT 1` use karte hain, toh result ek single sObject mein store kar sakte hain (List ki zaroorat nahi).

**Code Example:**

```java
// Top 10 accounts sorted by Name
Account[] accts = [SELECT Name, Phone FROM Account 
                   WHERE NumberOfEmployees > 25 
                   ORDER BY Name DESC 
                   LIMIT 10];

```

---

## 5. Apex Variables in SOQL (Bind Variables)

Apex variables ko SOQL query ke andar use karne ke liye variable ke pehle **Colon (:)** lagaya jata hai. Isse **Bind** kehte hain.

**Code Example:**

```java
String targetDepartment = 'Wingo';
Contact[] techContacts = [SELECT FirstName, LastName FROM Contact 
                          WHERE Department = :targetDepartment];

```

---

## 6. Querying Related Records

Salesforce mein Parent-Child relationship ke beech query karna easy hai.

### A. Parent to Child (Inner Query)

Parent record ke saath uske saare child records mangwane ke liye inner query use karte hain. Yahan **Relationship Name** (Contacts) use hota hai, object name nahi.

```java
Account[] accts = [SELECT Name, (SELECT FirstName, LastName FROM Contacts) 
                   FROM Account WHERE Name = 'SFDC Computing'];
// Child records access karna
Contact[] cts = accts[0].Contacts;

```

### B. Child to Parent (Dot Notation)

Child record se parent ki details nikalne ke liye Dot `.` use karte hain.

```java
Contact[] cts = [SELECT Account.Name FROM Contact 
                 WHERE FirstName = 'Carol' AND LastName = 'Ruiz'];
String parentName = cts[0].Account.Name;

```

* **Custom Objects:** Custom relationships ke liye `__r` suffix use hota hai (jaise `Line_Items__r`).

---

## 7. SOQL For Loops (Efficient Batching)

Heap size limit se bachne ke liye aur governor limits ko manage karne ke liye SOQL For Loop best hai. Ye data ko **200 records ke batches** mein process karta hai.

**Code Example:**

```java
// SObject List format (Best Practice)
for (Account[] tmp : [SELECT Id FROM Account WHERE Name LIKE 'for loop _']) {
    // Ye loop har 200 records ke batch par ek baar chalega
    System.debug('Batch size: ' + tmp.size());
}

```

---

## 8. Hands-on Challenge Solution (Logic)

Aapko ek class `ContactSearch` banani hai jisme method `searchForContacts` do strings accept kare.

**Solution Code:**

```java
public class ContactSearch {
    public static List<Contact> searchForContacts(String lastName, String postalCode) {
        // Querying Contact based on parameters
        List<Contact> conList = [SELECT Id, Name FROM Contact 
                                 WHERE LastName = :lastName 
                                 AND MailingPostalCode = :postalCode];
        return conList;
    }
}

```

# **Notes: Salesforce Object Search Language (SOSL)**

### **1. Introduction to SOSL**

* **Definition:** SOSL ek Salesforce search language hai jiska use records ke andar text search karne ke liye kiya jata hai.
* **Multiple Objects:** SOSL ki sabse badi khoobi yeh hai ki yeh ek saath kai standard aur custom objects ke fields ko search kar sakta hai.
* **Inspiration:** Yeh Apache Lucene par based hai.
* **Inline SOSL:** Jab SOSL queries ko directly Apex code ke andar likha jata hai, toh use "Inline SOSL" kehte hain.

### **2. SOSL vs SOQL (Key Differences)**

| Feature | SOQL | SOSL |
| --- | --- | --- |
| **Object Scope** | Sirf ek standard ya custom object ko query karta hai. | Ek saath kai objects ko search kar sakta hai. |
| **Matching Type** | Default mein "Exact Match" karta hai. | "Word Match" par kaam karta hai (e.g., 'Digital' search karne par 'The Digital Company' bhi mil jayega). |
| **Use Case** | Jab aapko pata ho ki data kis object mein hai aur specific criteria chahiye. | Jab aapko multiple objects mein text search karna ho aur exact location na pata ho. |
| **Fields** | Aapko fields specify karni padti hain. | Yeh zyadatar text fields ko search kar sakta hai. |

---

### **3. Prerequisites (Sample Data Setup)**

Agar aapko SOSL test karna hai, toh Developer Console ke **Execute Anonymous** window mein niche diya gaya code run karein:

```java
// Account aur related Contact add karna
Account acct = new Account(
    Name='SFDC Computing',
    Phone='(415)555-1212',
    NumberOfEmployees=50,
    BillingCity='San Francisco');
insert acct;

ID acctID = acct.ID; // Inserted ID lena

Contact con = new Contact(
    FirstName='Carol',
    LastName='Ruiz',
    Phone='(415)555-1212',
    Department='Wingo',
    AccountId=acctID);
insert con;

// Bina contact wala Account add karna
Account acct2 = new Account(
    Name='The SFDC Query Man',
    Phone='(310)555-1213',
    NumberOfEmployees=50,
    BillingCity='Los Angeles',
    Description='Expert in wing technologies.');
insert acct2;

```

---

### **4. Basic SOSL Syntax**

SOSL likhne ka tarika **Apex** aur **Query Editor/API** mein thoda alag hota hai:

* **In Apex:** Search query single quotes (`' '`) mein hoti hai.
`FIND 'SearchQuery' [IN SearchGroup] [RETURNING ObjectsAndFields]`
* **In Query Editor/API:** Search query curly brackets (`{ }`) mein hoti hai.
`FIND {SearchQuery} [IN SearchGroup] [RETURNING ObjectsAndFields]`

#### **Syntax Components:**

1. **SearchQuery:** Woh text jo aap search karna chahte hain (Single word ya Phrase).
* Wildcards: `*` (zero ya zyada characters) aur `?` (sirf ek character).
* Case-Insensitive: 'Customer', 'customer', aur 'CUSTOMER' sabka result same hoga.


2. **SearchGroup (Optional):** Yeh batata hai ki search kahan karni hai.
* `ALL FIELDS` (Default)
* `NAME FIELDS`
* `EMAIL FIELDS`
* `PHONE FIELDS`
* `SIDEBAR FIELDS`


3. **ObjectsAndFields (Optional):** Kaunsa data return karna hai (e.g., `RETURNING Account(Name), Contact(FirstName)`).

---

### **5. Search Examples & Logic**

| Search Term | Description | Result based on Sample Data |
| --- | --- | --- |
| **The Query** | Dono words search karega kisi bhi order mein. | Account: "The SFDC Query Man" |
| **Wingo OR Man** | Ya toh 'Wingo' mile ya 'Man'. | Contact: Carol Ruiz; Account: The SFDC Query Man |
| **1212** | Dash `-` ke baad wale numbers ko word maanta hai. | Account: SFDC Computing; Contact: Carol Ruiz |
| **wing*** | Wildcard search jo 'wing' se shuru hone wale sab search karega. | Account: The SFDC Query Man; Contact: Maria Ruiz |

---

### **6. SOSL Apex Example (List of Lists)**

SOSL hamesha `List<List<sObject>>` return karta hai kyunki isme multiple objects ka data hota hai.

```java
String soslFindClause = 'Wingo OR SFDC';
// Inline SOSL with Binding (:soslFindClause)
List<List<sObject>> searchList = [FIND :soslFindClause IN ALL FIELDS 
                    RETURNING Account(Name), Contact(FirstName, LastName, Department)];

// Data ko separate lists mein cast karna
Account[] searchAccounts = (Account[])searchList[0];
Contact[] searchContacts = (Contact[])searchList[1];

System.debug('Found the following accounts.');
for (Account a : searchAccounts) {
    System.debug(a.Name);
}

System.debug('Found the following contacts.');
for (Contact c : searchContacts) {
    System.debug(c.LastName + ', ' + c.FirstName);
}

```

---

### **7. Advanced SOSL (Filtering & Ordering)**

Aap `RETURNING` clause ke andar results ko refine kar sakte hain:

* **Filtering (WHERE):**
`RETURNING Account(Name, Industry WHERE Industry='Apparel')`
* **Ordering (ORDER BY):**
`RETURNING Account(Name, Industry ORDER BY Name)`
* **Limiting (LIMIT):**
`RETURNING Account(Name, Industry LIMIT 10)`
